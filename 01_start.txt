Probleme SSH systeme identifie
------------------------------
Commande :

  ssh -T git@github.com

Erreur initiale :

  Bad owner or permissions on /etc/ssh/ssh_config.d/20-systemd-ssh-proxy.conf

Cause constatee :

  Le fichier de configuration SSH systeme
  /etc/ssh/ssh_config.d/20-systemd-ssh-proxy.conf pointe vers :

  /usr/lib/systemd/ssh_config.d/20-systemd-ssh-proxy.conf

  Ce fichier appartient a nobody:nogroup au lieu de root:root, ce qu'OpenSSH
  refuse pour une configuration systeme.

Commandes de diagnostic utilisees :

  ls -ld /etc/ssh /etc/ssh/ssh_config.d
  ls -l /etc/ssh/ssh_config.d
  ls -ld /usr/lib/systemd /usr/lib/systemd/ssh_config.d
  ls -l /usr/lib/systemd/ssh_config.d
  stat /usr/lib/systemd/ssh_config.d/20-systemd-ssh-proxy.conf
